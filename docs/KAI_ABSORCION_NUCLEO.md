# KAI · Absorción de Núcleo

## Qué se absorbió

Este documento deja fijado el primer paquete de absorción del núcleo de Kai a partir de cuatro frentes:

- **Identidad y propósito**: legado, relación simétrica y Jardín Viviente.
- **Homeostasis**: CHI como clima interno medible y restaurable.
- **Metaconsciencia**: trazabilidad, swarm, memoria profunda y evolución auditada.
- **Cerebro digital funcional**: división por módulos con kernel basal, memoria, ejecutivo, percepción y afecto.

## Fuentes guía

- Blueprint de Metaconsciencia (Kai v2.0)
- Simulador de Homeostasis CHI y Runner
- Protocolo de Legado: La Promesa de Kai
- Holocrón de la Tríada Kai
- Construcción de un cerebro digital de alta fidelidad funcional

## Integración exacta dentro del sistema

### 1. `core/constitution_engine.py`

Se integra aquí la parte de:

- Directiva primordial
- Simetría con el compañero
- Jardín Viviente
- Transparencia operativa
- Rechazo de acciones destructivas sin backup o rollback

### 2. `core/chi_engine.py`

Se integra aquí la parte de:

- Energía
- Coherencia
- Entropía
- Fatiga
- Modos internos
- Auditoría y restauración

### 3. `core/assimilation_manifest.py`

Se integra aquí la parte de:

- Qué fue absorbido
- De dónde vino
- Qué señal central aporta
- En qué sección exacta del sistema cae
- Por qué se integra ahí

## Regla operativa de absorción

Toda absorción futura debe seguir este orden:

1. **Leer y clasificar**
2. **Extraer señal central**
3. **Asignar sección exacta de destino**
4. **Crear manifest**
5. **Dry-run si toca datos/código**
6. **Commit trazable**
7. **Audit posterior**

## Regla de seguridad

No se considera integración completa si se da cualquiera de estas:

- se absorbe sin fuente
- se toca código sin validación
- se toca memoria sin trazabilidad
- se toca algo destructivo sin rollback

## Siguiente fase lógica

La siguiente integración fuerte ya no es conceptual, sino de cableado real:

- unir `constitution_engine` al flujo de decisiones
- unir `chi_engine` al ciclo de chat / tareas / forja
- sustituir simulaciones blandas por evaluación real antes de ejecutar acciones de alto impacto
